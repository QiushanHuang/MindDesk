# MindDesk v2.3.0

MindDesk v2.3.0 is a minor product-logic release focused on project re-entry. It adds Workspace Resume Brief v0 and Home status badges so returning to a workspace surfaces the next useful signals without creating a dashboard or triggering external actions.

## Product Changes

- Added a compact Workspace Resume Brief below the workspace header.
- Added Home Recent Workspace badges for overdue tasks, due-soon tasks, open tasks, and resource issues.
- Summarized each workspace's next tasks, known resource issues, canvas card/link counts, and recent snippets.
- Kept the new UI read-only: resume controls only open a workspace or switch internal Workspace tabs.
- Capped Home summaries to the six recent workspaces and capped visible detail lists to avoid turning Home into a cross-workspace task list.

## Reliability And Safety

- Added a pure `MindDeskCore` re-entry policy with deterministic ordering, workspace-scoped visibility, and fail-closed handling for unknown scopes.
- Added app-layer mapping from SwiftData models into pure core records without changing the SwiftData schema.
- Counted dangling resource and snippet canvas references, including missing, private, and unknown-scope records.
- Stabilized next-task ordering so equal-priority tasks are not reshuffled by edit timestamps.
- Added large-workspace degradation: detail IDs are omitted while counts and badges remain available.

## Distribution

- Public artifacts should be built through the Developer ID notarized release workflow or an equivalent local Developer ID notarization run.
- Release workflow packaging can run from `main` or the matching `v2.3.0` tag.
- Draft GitHub Release publishing is tag-only and uses `--verify-tag`, so the workflow cannot create a version tag implicitly.
- Internal ad-hoc packages remain opt-in only with `--mode adhoc --allow-adhoc` and include an `-adhoc` artifact suffix.

## 中文摘要

MindDesk v2.3.0 是一次面向项目重入体验的小版本更新。新增 Workspace Resume Brief v0 和 Home 状态徽章，让用户回到复杂 workspace 时可以先看到下一步任务、资源问题、画布规模和最近使用的 snippets，而不需要额外打开 dashboard。

本轮主要结果：

- Workspace 顶部新增紧凑的 Resume Brief，用于展示 next tasks、known resource issues、canvas card/link counts 和 recent snippets。
- Home Recent Workspaces 新增最多两个状态徽章，帮助快速识别逾期任务、即将到期任务、打开任务和资源问题。
- 新 UI 保持只读，只会打开 workspace 或切换 Workspace 内部 tab，不会打开 Finder、Terminal、执行命令或触发 bookmark 授权。
- Core policy 采用 fail-closed 可见性策略，workspace-private 资源和 snippet 不会泄漏到其他 workspace brief。
- 大 workspace 会降级为 count-only 状态，避免在重入摘要里构建过多明细列表。
