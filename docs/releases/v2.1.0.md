# MindDesk v2.1.0

MindDesk v2.1.0 is a small product-logic release focused on trusted project re-entry. The work tightens the portable workspace model, makes resource references easier to reason about, and hardens import, canvas drop, and store recovery failure paths.

## Scope

| Area | v2.1.0 Scope |
| --- | --- |
| Manifest v2 portability | JSON exports now include workspace Todo groups and tasks, including details, completion state, pinned/order state, due dates, completion dates, and linked resources. |
| Manifest compatibility | v1 JSON manifests still decode and import with empty Todo collections, while v2 manifests gain stricter Todo reference validation. |
| Resource reference integrity | A shared ReferenceIndex and CleanupPlan describe resource usage across workspace resources, canvas cards, Todos, snippet working directories, and Finder aliases. |
| Metadata deletion safety | Removing a MindDesk resource clears related canvas nodes, Todo linked-resource fields, snippet working directories, and alias status without deleting Finder files. |
| Import and canvas drop feedback | Resource import tracks inserted, reused, skipped, failed, and over-limit outcomes; canvas drops skip resources already present on the same canvas. |
| Store recovery rollback | Publishing a restored SwiftData SQLite file set now attempts to roll quarantined original files back if the restored copy fails. |

## New and Improved

- Added Manifest schema v2 for portable Todo group and task export/import.
- Kept Manifest schema v1 import compatible by defaulting missing Todo collections to empty arrays.
- Excluded Todo records from Global Library Only exports so workspace-only task state is not leaked into a library-only package.
- Added Todo import validation for duplicate IDs, missing workspaces, missing groups, missing linked resources, and oversized text fields.
- Allowed empty canvas accent color values during manifest import so the default/reset color state round-trips cleanly.
- Added ResourceIdentity path normalization and ResourceKind classification primitives for explainable resource identity decisions.
- Added ReferenceIndex and CleanupPlan as a narrow adapter over current string references instead of introducing a risky storage-model rewrite.
- Wired resource deletion through CleanupPlan so linked Todos are cleared along with canvas nodes, snippet working directories, and alias status.
- Hardened ResourceImportService with a 200-item input cap, per-item skipped/failed reasons, duplicate-input detection, and partial success summaries.
- Updated Canvas drop behavior to avoid creating duplicate resource cards on the same canvas.
- Added rollback handling when publishing a recovered store fails after the existing SQLite file set has already been quarantined.
- Expanded core regression coverage around manifest v2, v1 compatibility, Todo references, reference cleanup, canvas drop dedupe, resource identity, import summaries, and default color round-trip.

## Distribution

- Public artifacts should be built through the Developer ID notarized release workflow or an equivalent local Developer ID notarization run.
- Release workflow packaging can run from `main` or the matching `v2.1.0` tag.
- Draft GitHub Release publishing is tag-only and uses `--verify-tag`, so the workflow cannot create a version tag implicitly.
- Internal ad-hoc packages remain opt-in only with `--mode adhoc --allow-adhoc` and include an `-adhoc` artifact suffix.

## 中文摘要

MindDesk v2.1.0 是一次小版本产品逻辑更新，重点是让项目状态更可信、资源引用更可信、恢复路径更可信。

这版补齐了几个实际使用中会影响长期项目管理的边界：

- JSON manifest 升级到 v2，导出和导入 Todo Group、任务详情、完成状态、DDL、排序和关联资源。
- v1 manifest 继续兼容，新 Todo 字段缺失时默认当作空列表处理。
- Global Library Only 导出不再带出 Workspace Todo，避免把项目任务状态混入资源库备份。
- 资源删除不触碰真实 Finder 文件，但会一致清理 MindDesk 内部的 Canvas、Todo、Snippet 和 Alias 引用。
- 批量导入最多处理 200 项，并明确记录每项 skipped / failed 的原因；Canvas 拖放不会在同一画布重复创建同一资源卡片。
- Store 恢复发布失败时会尝试回滚原 SQLite 文件集，降低恢复过程中产生部分 store 的风险。

正式公开发布包仍应通过 GitHub Release workflow 使用 Developer ID 签名、notarization 和 stapling 生成。
