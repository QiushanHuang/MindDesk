# MindDesk v3.0.0

MindDesk v3.0.0 focuses on AI-readable project handoff, safer agent review, portable interchange, richer workspace controls, and release-ready packaging guardrails. It keeps agents read-only by default: external tools can inspect curated context and draft proposal envelopes, but every side effect still requires Proposal Review plus explicit immediate in-app confirmation.

## Product Changes

- Added Agent Review package export with app-authored prompts, workflow guidance, custom guidance support, help topics, extension capability summaries, and readiness summaries.
- Added a normalized `.mip.json` interchange package wrapper around manifest data so humans and AI tools can inspect package metadata, validation reports, policies, context, references, help topics, and proposal contracts consistently.
- Added Proposal Review gate and review sheet surfaces for validating agent-generated proposal envelopes before any in-app side effect can proceed.
- Added typed portable manifest wire metadata while preserving legacy manifest import compatibility and rejecting unsupported typed versions.
- Added richer Settings, import, export, proposal, validation, and Agent Review presentation surfaces.
- Improved Canvas scalability with viewport-aware edge indexing, smoother routing policy controls, and large-workspace degradation paths.
- Added a Canvas Codex panel that prepares bounded read-only Canvas context and starts an in-sidebar `codex exec --json` session without opening Terminal.
- Added editable Canvas Codex prompt groups and presets for organization, review, summary, and Proposal Review-oriented workflows.
- Hardened the Canvas Codex sidebar session with an empty temporary Codex root, a `service_tier="flex"` override for incompatible local CLI config, bounded output retention, and stable prompt-template storage.
- Kept Canvas edge glow animation active during viewport panning and zooming while retaining geometry-edit performance safeguards.
- Integrated the v2.4.0 sibling release product behavior: Overview-first workspace entry, dedicated Tasks tab, lazy Canvas creation, and exact resource-removal cleanup messaging.
- Expanded Help content so human users and AI retrieval can find the agent workflow, extension capability search, proposal review, manifest format, and import/export safety boundaries.

## Reliability and Safety

- Agent packages remain read-only and non-authorizing; they do not contain security-scoped bookmark data, raw SQLite stores, backup archives, command output logs, directory listings, or file contents.
- Proposal Review validates raw package and proposal JSON data before creating pending review, including authority mirror drift, package policy drift, stale manifest context, replay attempts, unsupported operation payloads, and size limits.
- Validation reports use stable machine-readable diagnostics with redaction/tokenization for unsafe raw details.
- Custom guidance is explicitly untrusted, non-authoritative, capped before export, and unable to override help topics, policies, validation, capability catalogs, Proposal Review, or confirmation requirements.
- Release scripts now include worktree cleanliness checks, artifact verification, failure diagnostic preservation, ad-hoc smoke validation, and GitHub Release workflow guardrails.

## Distribution

- Public artifacts should be built through the Developer ID notarized release workflow or an equivalent local Developer ID notarization run.
- Release workflow packaging can run from `main` or the matching `v3.0.0` tag.
- Draft GitHub Release publishing is tag-only and uses `--verify-tag`, so the workflow cannot create a version tag implicitly.
- Internal ad-hoc packages remain opt-in only with `--mode adhoc --allow-adhoc` and include an `-adhoc` artifact suffix.

## Lineage

- `v3.0.0` is currently documented and packaged from the `codex/v3-foundation-p0` foundation branch.
- `v2.4.0` is a sibling release on `origin/codex/v2-4-c-lite`, not an ancestor of this branch.
- The v2.4.0 product behavior has been manually integrated into this branch for v3.0.0 release-line continuity.
- The preserved sibling release note is available at `docs/releases/v2.4.0.md`.
