# MindDesk v1.4.0

MindDesk v1.4.0 is a minor release that turns the image-based upgrade notes into a focused polish and repair pass. The SwiftPM package, executable, bundle identifier, app bundle, storage layout, release scripts, and visible app identity now use MindDesk naming, with a startup migration path from the previous store location.

## New and Improved

- Completed MindDesk naming for the SwiftPM package, targets, executable, bundle identifier, storage layout, release scripts, app bundle, and UI.
- Added startup migration from the previous local SwiftData store into the new MindDesk store path.
- Added Workspace sidebar drag reordering.
- Added Workspace reference cards, Web Page cards, and a fuller Cmd+K Quick Open command palette with ranked results, keyboard navigation, and empty states.
- Added inline task detail summaries that collapse whitespace and truncate on one line.
- Added a Single-use Connect setting in Canvas and Settings.
- Added responsive scrolling for the Canvas right rail.
- Lowered Canvas edge routing clearance so links do not detour around cards that are near but not intersecting.
- Added direct line selection so deleting a selected link no longer deletes every line attached to a card.
- Paused animated link flow during active Canvas interactions to reduce redraw pressure.
- Added deletion undo for Canvas cards/links and task-board tasks/groups, plus Cmd+L to start a link from the selected card, Shift+Cmd+L to connect the selected cards, and Shift+Cmd+R to reverse the selected link.
- Added Reverse Selected Link and Start Link From Card actions in the Canvas right rail, plus card context-menu actions for starting links from cards and frames.
- Reduced Canvas interaction churn by skipping obstacle routing while dragging, resizing, panning, or zooming; limiting animated link flow on dense canvases; hiding non-selected link handles during interaction; and coalescing scroll zoom / text edit persistence.
- Added Developer ID notarization guardrails to `script/package_release.sh`, plus GitHub Actions CI and a manual notarized release workflow.

## Validation

The release should be validated with:

- `swift test`
- `swift build`
- `git diff --check`
- `./script/build_and_run.sh --verify`
- `bash -n script/package_release.sh`
- `plutil -lint script/release.entitlements`
- GitHub Actions CI on the pushed branch.
- `./script/package_release.sh --mode notarized --identity "Developer ID Application: Qiushan Huang (TEAMID)" --team-id TEAMID --notary-profile minddesk-notary`
- `xcrun stapler validate MindDesk.app` and Gatekeeper `spctl` assessment on the final DMG before upload.
