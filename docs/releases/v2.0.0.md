# MindDesk v2.0.0

MindDesk v2.0.0 is the first major release built from the fully merged upgrade set. It keeps the visual-workbench model introduced in v1.x, but raises the shipping baseline across resource safety, settings consistency, canvas responsiveness, and release operations so the product can be published as a coherent new version instead of another patch-level consolidation.

## Scope

| Area | v2.0.0 Scope |
| --- | --- |
| Canvas performance baseline | Carries forward the visible-work-only rendering, interaction-time lightweight drawing, frame drag snapshots, and density-aware controls introduced during the v1.4 performance passes. |
| Import and export hardening | Global and workspace-scoped manifest records are now validated more strictly, broken or mismatched references are rejected earlier, and import paths preserve scope boundaries instead of silently degrading. |
| Resource access safety | Bookmark persistence is fail-closed, folder preview enumeration is bounded, drag/drop targets are filtered by destination type, and command snippets reject invalid working directories instead of falling back without notice. |
| Settings consistency | App preference defaults are centralized, obsolete keys are clearable, and the Settings UI can restore the full default configuration across Home, Canvas, Todo Board, and related app surfaces. |
| Geometry and interaction guards | Non-finite viewport geometry is sanitized, hit-testing and screen-projection logic are safer, and shared policy helpers now cover working directories, folder preview scan limits, and resource drop targets. |
| Release operations | README, release notes, package naming, metadata checks, scripts, and GitHub release workflow references are aligned around the formal `v2.0.0` release line. |

## New and Improved

- Promoted the merged upgrade branch into a major-version release line with versioned documentation and packaging metadata for `v2.0.0`.
- Preserved the large-canvas responsiveness work from v1.4: visibility-aware routing, lightweight card rendering during active interaction, density-aware handles and controls, and steadier frame dragging through drag-start snapshots.
- Added centralized `AppPreferenceDefaults` so `@AppStorage` behavior stays aligned across Home, Canvas, Todo Board, command/snippet settings, and other preference-driven UI.
- Added Reset All Settings support in Settings so stale or experimental app preferences can be restored to a known-good baseline without manual key cleanup.
- Hardened manifest validation so global-scope records cannot carry workspace IDs and scope mismatches are rejected earlier.
- Restored drop-target filtering for pinned resources and resource lists so incompatible drag destinations are skipped with clearer status feedback.
- Tightened command-snippet working-directory behavior so an invalid configured folder reference raises an error instead of silently falling back to an unintended path.
- Made bookmark creation fail closed during resource import, preventing partial resource entries from being stored with missing or invalid security-scoped bookmark data.
- Bounded folder preview scanning before sorting so large directories no longer force unbounded enumeration just to show a preview list.
- Consolidated shell quoting for Terminal launch commands so working-directory changes and quoted command execution share one safe path builder.
- Added non-finite geometry sanitization and related regression coverage to keep canvas viewport projection and screen-space calculations from drifting into undefined states.
- Preserved and extended regression coverage for performance policy boundaries, import validation, working-directory policy, resource drop targets, folder preview scan limits, settings restore behavior, and shell quoting.
- Kept the release guardrails in place for CI, metadata verification, entitlements linting, script syntax checks, and tag-based GitHub Release packaging.

## Distribution

- Public artifacts should be built through the Developer ID notarized release workflow or an equivalent local Developer ID notarization run.
- Release workflow packaging can run from `main` or the matching `v2.0.0` tag.
- Draft GitHub Release publishing is tag-only and uses `--verify-tag`, so the workflow cannot create a version tag implicitly.
- Internal ad-hoc packages remain opt-in only with `--mode adhoc --allow-adhoc` and include an `-adhoc` artifact suffix.

## Validation

This release was validated with:

- `swift build`
- `swift test`
- `./script/build_and_run.sh --verify`
- `./script/verify_release_metadata.sh`
- `bash -n script/build_and_run.sh`
- `bash -n script/package_release.sh`
- `bash -n script/verify_release_metadata.sh`
- `git diff --check`
- `plutil -lint script/release.entitlements`

The local environment used for this release does not currently have a Developer ID signing identity or `minddesk-notary` keychain profile, so the formal notarized package must be produced by `.github/workflows/release.yml` from the pushed `v2.0.0` tag with repository signing and notarization secrets configured.

## 中文摘要

MindDesk v2.0.0 是基于完整合并后的升级集形成的首个正式大版本。它没有推翻 v1.x 已有的可视化工作台模型，而是把资源访问安全、设置一致性、大画布响应速度和发布链路整体抬到一个可以正式发布的新基线。

这版的重点不是“再加一个零散功能”，而是把几类容易出问题的边界补齐：

- 大画布继续沿用 v1.4 的性能专项，包括按视口裁剪昂贵渲染、交互态轻量卡片绘制、密集画布下的控制点/动画限流，以及更稳定的 Frame 拖拽快照。
- 导入导出更严格：global 和 workspace scope 的 manifest 记录不再允许混淆，错误引用会更早被拒绝，而不是导入后再隐性漂移。
- 资源访问链路更保守：bookmark 持久化失败会直接中止，文件夹预览扫描有上限，拖放目标按目的地类型过滤，命令 snippet 的工作目录无效时会明确报错。
- 设置系统更统一：集中默认值、可清理废弃 key，并在 Settings 中提供一键恢复默认设置能力，让 Home、Canvas、Todo Board 和相关界面的偏好行为一致。
- 发布信息与打包链路已经整体切到 `v2.0.0`，包括 README、版本文档、脚本校验、CI 守卫和 GitHub Release workflow。

当前本机环境缺少 Developer ID 签名证书和 `minddesk-notary` 凭据，因此正式 notarized 包需要从推送后的 `v2.0.0` tag 通过 GitHub Release workflow 生成。
