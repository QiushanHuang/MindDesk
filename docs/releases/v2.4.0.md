# MindDesk v2.4.0

Lineage note: `v2.4.0` is a sibling release on `origin/codex/v2-4-c-lite`. It is not an ancestor of the current `codex/v3-foundation-p0` branch. Keep this release note as historical release record; the v2.4.0 product behavior was later manually integrated into the current v3 branch during release readiness closeout.

MindDesk v2.4.0 is a minor product-logic release focused on reducing unnecessary Canvas work and making resource removal safer to understand. It introduces an Overview-first workspace entry, a dedicated Tasks tab, lazy Canvas creation, and exact metadata cleanup messaging for resource removal.

## Product Changes

- Workspace detail now defaults to Overview instead of Canvas.
- Switching workspaces resets the detail area back to Overview.
- Tasks now has a dedicated full-height tab, while the Canvas task board remains a compact panel.
- New workspaces and seed data no longer create Canvas records up front.
- Canvas records are created only when the user explicitly opens the Canvas tab.
- The Overview "Next" action now opens the Tasks tab instead of showing task titles inline.

## Reliability And Safety

- Resource removal now snapshots the cleanup plan used for both the confirmation message and destructive action.
- The confirmation message lists exact MindDesk metadata impacts: Canvas cards, Canvas links, todo linked-resource fields, command working directories, and alias records.
- Removing a resource deletes incident Canvas links as well as its resource cards.
- Resource removal remains metadata-only: Finder files and folders are not deleted, moved, or renamed.
- Existing Canvas lookup is capped with `fetchLimit = 1` and covered with in-memory SwiftData tests.
- No SwiftData schema, import/export schema, or system service changes are included.

## Distribution

- This release is packaged as a local ad-hoc draft release, matching the previous draft release style.
- The app and DMG are not Developer ID notarized in this build.
- The local ad-hoc artifacts include an `-adhoc` suffix.
- Release packaging uses `./script/package_release.sh --mode adhoc --allow-adhoc`.
- Draft GitHub Release publishing uses the pushed matching `v2.4.0` tag.
