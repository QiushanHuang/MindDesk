# MindDesk v2.2.0

MindDesk v2.2.0 is a maintenance and polish release for existing product behavior. It does not add a new product surface. The focus is interaction predictability, workspace boundary safety, calmer defaults, and performance/stability guardrails for the Canvas, Workspace, Settings, Tasks, and release workflow.

## Audit Log

| Area | Issue Found | Fix | Result |
| --- | --- | --- | --- |
| Canvas mouse and trackpad input | Horizontal trackpad scroll and tiny wheel events could be swallowed by the scroll-zoom monitor. | Added a wheel-event policy that zooms only on finite, meaningful, vertical-dominant deltas. | Horizontal scrolling is allowed to pass through instead of being consumed by zoom logic. |
| Canvas resize interaction | Resize handle hit areas scaled directly with zoom, becoming too small at low zoom and too large at high zoom. | Added clamped resize handle hit sizing and wired it into the actual SwiftUI resize control. | Resize handles remain reachable across zoom levels without excessive overlap. |
| Canvas edge editing | Edge control handles could disappear during drag start because dense/interacting render policy hid them. | Edge handle visibility now treats active dragging as an explicit visible state. | Control handles stay visible while a bend point is being adjusted. |
| Canvas rendering cost | Small drags and pans previously reduced peer card details too aggressively, while dense canvases needed stronger throttling. | Detail reduction now keeps richer peer details for sparse spatial movement and reduces detail only when spatial interactions exceed the density threshold or geometry work is active. | Small boards feel less visually jumpy, while dense boards still lower render pressure. |
| Canvas transient state | Selected, editing, connection, drag, and resize state could point at nodes that were removed by import or deletion flows. | Added node-state reconciliation and used it when node IDs change. Link creation also rejects missing endpoints. | Canvas state no longer holds dangling node references after model changes. |
| Workspace resource boundaries | Workspace-private resources could appear in another workspace's Canvas add menu. | The Canvas add menu now exposes only Global Library resources and resources owned by the current workspace. | Workspace-private resources stay private to their workspace while global resources remain reusable. |
| Manifest validation | Cross-workspace workspace-scoped resource and snippet references could enter through imported canvas/snippet records. | Import validation rejects workspace-scoped working directories, resource nodes, and snippet nodes from another workspace while allowing global records. | Imported metadata cannot silently create private cross-workspace references. |
| Home recency | Recent Workspaces used sidebar ordering rather than actual recent-open history. | Home now orders recent workspaces by `lastOpenedAt`, then `updatedAt`, then stable sidebar order. | Recently opened workspaces reappear where users expect them. |
| Workspace open metadata | Opening a workspace updated `updatedAt`, making a read-only navigation action look like an edit. | Workspace open now updates `lastOpenedAt` only. | Project metadata reflects content edits rather than navigation. |
| Task defaults | Opening a workspace could create an empty default task group, and the Canvas task panel opened by default. | Viewing a workspace no longer creates a group automatically, and the Canvas task panel default is closed. | Canvas first-open state is quieter and avoids unwanted task metadata. |
| Task terminology | UI and docs still used legacy deadline wording. | Current UI/docs use Due Date terminology. | Task wording is more standard and easier to scan. |
| Persisted geometry | Non-finite or extreme card sizes could reach layout. | Card-size policy now rejects non-finite, non-positive, and extreme dimensions. | Bad persisted geometry is contained before SwiftUI layout. |

## Subagent Review

Multiple gpt-5.5/xhigh audit passes were used for product logic, Canvas input, Settings/defaults, performance/stability, and release readiness. A follow-up code review found two concrete integration gaps:

- Resize hit-area clamping was tested but not wired into the real UI control.
- Canvas import validation covered workspace resources but initially missed workspace-scoped snippet nodes.
- Workspace recency could remain stale when SwiftUI reused the workspace detail view across direct workspace switches.
- Canvas switch handling did not yet clear all viewport, zoom, selection, and edge-control transient state.

All findings were fixed before release documentation was finalized, with core policy coverage where the behavior can be exercised outside SwiftUI lifecycle.

## New and Improved

- Home Recent Workspaces now uses actual last-opened recency.
- Opening a workspace no longer changes the workspace `updatedAt` timestamp.
- Workspace Resources view now keeps current-workspace resources separate from global reusable resources.
- Canvas Add Resource menu now offers Global Library and Current Workspace only.
- Canvas task panel starts closed by default.
- Viewing a workspace no longer creates an empty task group.
- Scroll-wheel zoom ignores horizontal or near-zero wheel events.
- Edge control handles remain visible during active dragging.
- Resize handle hit targets are clamped between usable minimum and maximum sizes.
- Canvas transient node state is reconciled when the node set changes.
- Link creation refuses missing source or target nodes.
- Dense Canvas spatial interactions reduce peer details only past the density threshold.
- Persisted card sizes reject non-finite and extreme dimensions.
- Manifest import validation rejects cross-workspace private resource and snippet references while preserving global reuse.
- Settings and docs use "Tasks" and "Due Date" wording consistently.

## Distribution

- Public artifacts should be built through the Developer ID notarized release workflow or an equivalent local Developer ID notarization run.
- Release workflow packaging can run from `main` or the matching `v2.2.0` tag.
- Draft GitHub Release publishing is tag-only and uses `--verify-tag`, so the workflow cannot create a version tag implicitly.
- Internal ad-hoc packages remain opt-in only with `--mode adhoc --allow-adhoc` and include an `-adhoc` artifact suffix.

## 中文摘要

MindDesk v2.2.0 是一次已有功能维护和体验打磨版本，没有新增产品入口。重点修正 Canvas 鼠标和触控板交互、Workspace 私有资源边界、默认展开状态、任务面板、Manifest 导入安全，以及大画布下的性能和稳定性守卫。

本轮主要结果：

- Canvas 横向滚动不再被缩放监听吞掉，resize 命中区在缩放后保持可用，拖动连线控制点时控制柄保持可见。
- Home Recent Workspaces 改为按最近打开排序，打开 workspace 不再把 `updatedAt` 标记为内容编辑。
- Canvas 只允许添加 Global Library 和当前 workspace 的资源，Manifest 导入会阻止跨 workspace 私有 resource/snippet 引用。
- Canvas 任务面板默认关闭，仅查看 workspace 不再自动创建空 Task Group。
- 大画布交互、节点消失、异常卡片尺寸和缺失连线端点都有更明确的保护。
- README、功能回归清单和发行说明已同步到 v2.2.0。

正式公开发布包仍应通过 GitHub Release workflow 使用 Developer ID 签名、notarization 和 stapling 生成。
