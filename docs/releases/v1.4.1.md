# MindDesk v1.4.1

MindDesk v1.4.1 is a small-version release that consolidates three parallel upgrade threads into one GitHub package. It keeps the v1.4 product surface intact while documenting the interaction upgrades, reducing CPU-heavy work during dense-canvas interaction, tightening migration/recovery behavior, and preserving the release guardrails needed for packaged distribution.

## Upgrade Thread Coverage

| Thread | Included Scope |
| --- | --- |
| `019e0bec...` | v1.4 interaction upgrade: Redo, Reverse Selected Link, Start Link From Card, Canvas smoothness work, and release documentation. |
| `019e0efb...` | Migration and recovery strategy: 30-minute startup backup throttling, `.complete` backup markers, `Quarantine/` isolation, staging restore, and SQLite trio recovery. |
| `019e0c2f...` | Three responsiveness and release-guard passes: Quick Open, Todo rename/editing, Canvas text redraw reduction, release metadata checks, and CI. |

## Scope

| Area | v1.4.1 Scope |
| --- | --- |
| Interaction upgrade | Redo, Reverse Selected Link, Start Link From Card, direct link operations, and updated release notes are documented as part of the v1.4 interaction pass. |
| Canvas responsiveness | Reduced work during drag, pan, zoom, resize, frame movement, and edge-control editing. |
| Visibility culling | Render work now focuses on visible cards and relevant edges instead of repeatedly routing and drawing off-screen geometry. |
| Obstacle routing | Obstacle routing is limited to small visible workloads and disabled while the canvas is actively interacting. |
| Link animation | Flow animation now respects visible edge count, visible card count, route-point complexity, zoom level, reduce-motion, and interaction state. |
| Card rendering | Dense or active views use lightweight card rendering, while selected and edited cards keep full detail. |
| Handles and controls | Passive resize handles and passive edge controls are hidden on dense canvases; selected, dragged, locked, and custom-control states remain discoverable. |
| Frame dragging | Frame drag now moves contained cards and link bend points from drag-start snapshots so undo data stays stable and live scans are avoided. |
| Quick Open and Todo responsiveness | Quick Open ranking/selection behavior, Todo rename/editing responsiveness, and Canvas text redraw reduction are covered by the response-speed pass. |
| Layout hardening | Canvas layout guards against duplicate IDs, non-finite coordinates, invalid sizes, and invalid spacing. |
| Import hardening | Manifest validation now rejects empty or duplicate IDs, cross-canvas parent and edge references, invalid web URLs, unsupported alias source types, and broken object references. |
| Migration and recovery | Startup backup throttling, `.complete` backup publication, `Quarantine/`, staging restore, and SQLite `.store` / `-shm` / `-wal` recovery are part of the release scope. |
| Resource access | Folder preview, Quick Look, and reauthorization paths now keep security-scoped access and rollback behavior tighter. |
| Release metadata | README, version metadata, release notes, metadata validation, and CI now point to v1.4.1 artifacts and GitHub Release packaging. |

## New and Improved

- Documented the v1.4 interaction upgrade set: Redo, Reverse Selected Link, Start Link From Card, Canvas smoothness work, and release notes.
- Added `CanvasPerformancePolicies` for animation, card-detail, resize-handle, edge-control, viewport-overscan, and active-edge rendering decisions.
- Deduplicated Canvas nodes and edges before snapshot rendering to avoid dictionary traps and repeated work from duplicated IDs.
- Added viewport-aware card and edge filtering before expensive route planning.
- Limited obstacle routing to visible card obstacles and visible candidate edges.
- Kept selected edges, transient control-point edges, and frame-moved bend-point edges visible during geometry interaction.
- Lowered passive resize-handle and passive edge-control density on large canvases.
- Added lightweight card rendering for dense or active canvas states, while preserving full rendering for selected and edited cards.
- Added native text-measurement fast paths and per-view title fitting caches for repeated Canvas chrome drawing.
- Reworked frame-drag bend-point movement to use drag-start snapshots keyed by edge ID.
- Kept Quick Open, Todo rename/editing, and Canvas text redraw behavior within the documented response-speed pass.
- Documented the migration/recovery strategy: 30-minute startup backup throttling, hidden incomplete folders, `.complete` markers, `Quarantine/`, staging restore, and SQLite trio recovery for `MindDesk.store`, `MindDesk.store-shm`, and `MindDesk.store-wal`.
- Preserved release guardrails through `script/verify_release_metadata.sh`, shell-script syntax checks, entitlements linting, `git diff --check`, CI on `main` / `codex/**`, and tag-only draft GitHub Release publishing.
- Added regression tests for performance policy boundaries, active-edge rendering, locked/custom controls, obstacle-routing thresholds, duplicate layout IDs, import validation, shell quoting, and security-scoped resource behavior.

## Distribution

- Public artifacts should be built through the Developer ID notarized release workflow or an equivalent local Developer ID notarization run.
- Release workflow packaging can run from `main` or the matching `v1.4.1` tag.
- Draft GitHub Release publishing is tag-only and uses `--verify-tag`, so the workflow cannot create a version tag implicitly.
- Internal ad-hoc packages remain opt-in only with `--mode adhoc --allow-adhoc` and include an `-adhoc` artifact suffix.

## 中文摘要

MindDesk v1.4.1 是把三个并行升级线程合并后的完整小版本：

| 线程 | 已纳入范围 |
| --- | --- |
| `019e0bec...` | v1.4 交互升级：Redo、反转连线、Start Link From Card、Canvas 流畅度专项和发布说明。 |
| `019e0efb...` | 迁移/恢复策略：30 分钟启动备份节流、`.complete` marker、`Quarantine/` 隔离、staging restore 和 SQLite 三件套恢复。 |
| `019e0c2f...` | 三轮响应速度和发布守卫：Quick Open、Todo rename/editing、Canvas 文本重绘优化、发布校验和 CI。 |

这版重点是降低大画布在拖拽、缩放、平移、拉伸、Frame 移动和连线控制点编辑时的 CPU 压力，减少不可见卡片、不可见连线、被动控制点、连线避让和动画带来的重复计算。

这次升级保持数据结构保守：新增的是 Canvas 性能策略、视口可见性判断、Frame 拖拽快照、交互中的轻量卡片绘制，以及覆盖这些边界的回归测试。同时合入 manifest import 校验、自动布局、Shell quoting、安全作用域资源访问、迁移/恢复策略和发布元数据加固，方便作为完整新版推送并发布到 GitHub。
